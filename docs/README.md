# 文档说明

本目录用于存放项目说明、设计文档、使用文档和约定说明。

## 建议组织方式

- `docs/<项目>/`：按具体子项目或模块拆分文档。
- `docs/<主题>/`：按主题归档，例如架构、部署、规范、测试。
- `docs/codex/`：记录 Codex 项目规则和项目内 skill 的使用方式。

## 当前建议内容

- 目录布局说明：记录 `AGENTS.md`、`.codex/skills/`、`scripts/`、`docs/`、`Bin/`、`temp/` 的用途。
- 规则使用说明：记录 `AGENTS.md` 和 `.codex/skills/` 中各规则的职责。
- 项目初始化说明：记录如何从本仓库派生新项目。

## 维护建议

- 本模板仓库的根目录 `README.md` 保留仓库总览和使用入口。
- 派生项目的根目录 `README.md` 可按项目需要自由改写，用于项目介绍、安装、使用或对外说明。
- 开发规范、命名规则、目录边界、验证流程和重复工作流优先写入 `AGENTS.md` 或 `.codex/skills/`；详细说明可放入 `docs/`。
- 新增文档时优先复用已有主题目录，仅在确有新主题时再新增一级目录。
