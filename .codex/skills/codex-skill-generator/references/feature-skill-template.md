## 修改流程

1. 读取功能入口和调用方。
2. 确认数据流、失败路径和边界条件。
3. 修改实现，并同步更新类型、配置、文档、测试和任务入口。
4. 删除被替换的旧流程和用户可见字符串。
5. 运行最小必要验证。

- 根目录 `README.md` 可描述功能使用方式；开发边界和维护规则以本 skill 为准。
